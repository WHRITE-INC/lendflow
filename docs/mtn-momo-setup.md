# MTN MoMo setup

## Sandbox

1. Create and verify an account at <https://momodeveloper.mtn.com/signup>.
2. Open <https://momodeveloper.mtn.com/product>, select **Collections**, and subscribe.
3. In your profile subscriptions, reveal either the Collections Primary or Secondary key.
4. Put that value in `.env` as `MTN_MOMO_COLLECTION_SUBSCRIPTION_KEY`.
5. Run `npm run mtn:setup`.

The setup command provisions a sandbox API user and API key through MTN's provisioning API and stores them in `.env`. The sandbox target environment is `sandbox` and its test currency is `EUR`.

Request to Pay is asynchronous. This project stores MTN's reference ID and polls the status endpoint, so payment completion still works without a callback. To receive callbacks too, deploy the app on HTTPS, set `MTN_MOMO_CALLBACK_URL`, and rerun setup. The callback URL must use the same host registered with the API user.

## Production in Zambia

Production access is not generated by the sandbox setup command. Use MTN's **Go Live** process at <https://momodeveloper.mtn.com/golive>, select Zambia and Collections, complete business KYC, and sign the contract. MTN then creates the business collections account and provides production credentials and endpoint details.

Production configuration should use the values issued by MTN Zambia. The local currency is `ZMW`; do not reuse sandbox API users or keys. MTN Zambia publishes Collections fees of 1% to 4% per transaction, with the final rate determined during onboarding. Commercial questions can be sent to `momocommercial@mtn.com`.

## Environment variables

```dotenv
MTN_MOMO_BASE_URL=https://sandbox.momodeveloper.mtn.com
MTN_MOMO_COLLECTION_SUBSCRIPTION_KEY=
MTN_MOMO_COLLECTION_API_USER=
MTN_MOMO_COLLECTION_API_KEY=
MTN_MOMO_TARGET_ENVIRONMENT=sandbox
MTN_MOMO_CURRENCY=EUR
MTN_MOMO_CALLBACK_URL=
```

Never expose these values through a `VITE_` variable or commit `.env`.
